# Zero 0.26

## Installation

```bash
npm install @rocicorp/zero@0.26
```

## Features

- [**Schema Backfill:**](/docs/schema#backfill) Zero now natively supports adding columns with non-constant defaults, or adding existing columns to a custom publication. There is no longer a need to touch every row in the database to backfill data manually.
- [**Scalar Subqueries:**](/docs/zql#scalar-subqueries) Added a new optional optimization for `exists` queries that filter by related data with a one-to-one relationship. This can improve query performance significantly for cases where the subquery result changes rarely.
- [**Virtual Scroll:**](/docs/status) Added [zero-virtual](https://github.com/rocicorp/zero-virtual) library for efficient infinite scrolling in React.
- [**Support for `time` and `timetz` columns:**](/docs/postgres-support#column-types) Added support for `time` and `timetz` columns (thanks [GRBurst](https://github.com/rocicorp/mono/pull/5417)).
- [**Comparing to `undefined`:**](/docs/zql#comparing-to-undefined) Added a new convenience for comparing to `undefined`. This is useful for filtering by nullable fields.
- [**zero.delete():**](/docs/auth#logging-out) Added a more convenient way to delete all data from the browser's IndexedDB database.

## Breaking Changes

Just two very minor breaking changes this time, that are unlikely to affect most users:

1. If you send custom headers to query or mutate endpoints, you must now allowlist them via [`ZERO_MUTATE_ALLOWED_CLIENT_HEADERS`](/docs/zero-cache-config#mutate-allowed-client-headers) or [`ZERO_QUERY_ALLOWED_CLIENT_HEADERS`](/docs/zero-cache-config#query-allowed-client-headers). By default, no client-provided headers are forwarded.
1. If you have very large mutations, note that WebSocket messages are now limited to 10MB by default. Increase `ZERO_WEBSOCKET_MAX_PAYLOAD_BYTES` if needed.
