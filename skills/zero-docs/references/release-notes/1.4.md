# Zero 1.4

## Installation

```bash
npm install @rocicorp/zero@1.4
```

## Upgrading

### userID: "anon"

If you are using `userID: 'anon'` for logged-out users, change it to `userID: null` or `userID: undefined`. Passing `userID: 'anon'` is being deprecated in advance of upcoming security work in Zero 1.5.

Change this:

**React**

```tsx
import {ZeroProvider} from '@rocicorp/zero/react'

return (
  <ZeroProvider userID={userID ?? 'anon'}>
    <App />
  </ZeroProvider>
)
```

**SolidJS**

```tsx
import {ZeroProvider} from '@rocicorp/zero/solid'

return (
  <ZeroProvider userID={userID ?? 'anon'}>
    <App />
  </ZeroProvider>
)
```

**TypeScript**

```ts
import {Zero} from '@rocicorp/zero'
import type {ZeroOptions} from '@rocicorp/zero'

const zero = new Zero({
  userID: userID ?? 'anon'
  // ...
})
```

To this:

**React**

```tsx
import {ZeroProvider} from '@rocicorp/zero/react'

return (
  <ZeroProvider userID={userID}>
    <App />
  </ZeroProvider>
)
```

**SolidJS**

```tsx
import {ZeroProvider} from '@rocicorp/zero/solid'

return (
  <ZeroProvider userID={userID}>
    <App />
  </ZeroProvider>
)
```

**TypeScript**

```ts
import {Zero} from '@rocicorp/zero'
import type {ZeroOptions} from '@rocicorp/zero'

const zero = new Zero({
  userID
  // ...
})
```

## Features

* [**Kysely Adapter:**](../server-zql.md#creating-a-database) Added `zeroKysely`, so Zero server mutators can run against a Kysely transaction.
* [**Analyze Query Library:**](../debug/slow-queries.md) Added `@rocicorp/zero/analyze`, for easier CLI-based query analysis.
* [**Shadow Sync:**](../zero-cache-config.md#shadow-sync-enabled) `zero-cache` can now periodically test initial sync in the background, to catch schema drift and full-resync failures earlier.
* [**Client Log Sink:**](https://github.com/rocicorp/mono/pull/5835) Added a `logSink` param to `Zero` constructor for redirecting Zero client logs.
* [**Cloud Zero Ad 🙃:**](https://zero.rocicorp.dev/cloud) Added a small notice about Cloud Zero in the `zero-cache-dev` startup output. Disable with `ZERO_ENABLE_STARTUP_MESSAGE=0`.

## Performance

* [7x faster client-side index construction](https://github.com/rocicorp/mono/pull/5844)
* [2x faster backfills by switching zero-cache to binary COPY](https://github.com/rocicorp/mono/pull/5789)
* Various optimizations to query hydration and updates:
  * [+9-16% hydration and +3-18% updates by reducing polymorphism](https://github.com/rocicorp/mono/pull/5801)
  * [+8-14% updates by reducing allocations](https://github.com/rocicorp/mono/pull/5843)
  * [+5-16% hydration by reducing allocations](https://github.com/rocicorp/mono/pull/5834)
  * [+7% hydration for queries with joins](https://github.com/rocicorp/mono/pull/5799)
* [Avoid full table scans from reordered compound primary keys](https://github.com/rocicorp/mono/pull/5807)
